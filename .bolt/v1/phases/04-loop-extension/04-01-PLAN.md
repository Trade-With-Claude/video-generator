# Phase 4 Plan: Loop Extension

## Acceptance Criteria

### AC1: Loop extends to target duration
- **Given** a 5-second rendered loop
- **When** I extend it to 60 seconds
- **Then** the output MP4 is exactly 60 seconds, 1080p30, H.264

### AC2: No visible seam at loop boundary
- **Given** a seamless 5-second loop extended to 15 seconds
- **When** I check the frames around the 5s boundary
- **Then** ffprobe confirms continuous stream with no discontinuity

### AC3: Full-length mode skips extension
- **Given** a render configured for full-length (no loop)
- **When** I render 10 seconds directly
- **Then** the output is 10 seconds with no intermediate loop file

## Tasks

### Task 1: Loop extension via FFmpeg stream copy
- **files:** `src/video_generator/loop.py`, `src/video_generator/config.py`
- **action:** `extend_loop(loop_path, output_path, target_duration, settings)` function. Uses FFmpeg's `-stream_loop` to repeat the short loop MP4 to fill the target duration, then `-t` to trim to exact length. Stream-copies (no re-encode) for speed. Also add `target_duration` field to Settings (default 300.0 = 5 min).
- **verify:** `python3 -c "from video_generator.loop import extend_loop; print('import ok')"`
- **done:** true

### Task 2: Integration test — 5s loop extended to 60s
- **files:** `scripts/test_loop.py`
- **action:** Script that renders a 5-second ambient loop, then extends it to 60 seconds via `extend_loop()`. Also tests full-length mode by rendering 10 seconds directly (no extension). Validates both output durations with ffprobe.
- **verify:** `python3 scripts/test_loop.py` produces two MP4s with correct durations
- **done:** true
